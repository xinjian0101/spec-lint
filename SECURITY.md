# Security Policy

Spec Lint is in the foundation phase and has not published a production-ready release.

Do not attach private specifications or sensitive project documents to public Issues. Use GitHub private reporting when available. Otherwise open a minimal Issue requesting a private communication channel.

Reports should identify the affected commit, input type, configuration, reproduction steps, observed impact, and a sanitized example.

Sensitive areas include path handling, configuration loading, report output, external link processing, plugin boundaries, and unexpected disclosure of document content.

The tool should operate on explicitly selected files, produce predictable reports, avoid changing source documents, and explain any optional external access.

Validated reports will be reviewed privately where practical. Public notes should identify affected versions and corrections without exposing user documents.
